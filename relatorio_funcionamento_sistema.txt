1. OBJETIVO GERAL DO SISTEMA
----------------------------------------------------------------------
O sistema foi desenvolvido para controlar de forma organizada a ida dos alunos ao banheiro durante as aulas. Ele evita que múltiplos alunos saiam ao mesmo tempo, criando uma fila de espera automática e registrando o histórico de duração de cada saída.


2. ARQUITETURA E FLUXO DE TRABALHO (WORKFLOW)
----------------------------------------------------------------------
O sistema funciona na arquitetura Cliente-Servidor utilizando uma API RESTful nativa (PHP) e consumida via Fetch API pelo Frontend (JavaScript Vanilla).

O fluxo principal é o seguinte:
a) Entrada de Dados: O professor/monitor insere o ID do aluno no painel principal e clica em "Confirmar".
b) Requisição (POST): O arquivo `js/app.js` envia uma requisição AJAX para `api/registro.php` com o ID informado.
c) Lógica de Negócio (API):
   - Verifica se o aluno existe no banco de dados (`models/Aluno.php`).
   - Verifica se já existe um aluno fora de sala (`models/Registro.php`).
   - Se o banheiro estiver LIVRE: O aluno sai e um registro "EM_ANDAMENTO" é criado.
   - Se o banheiro estiver OCUPADO: O aluno entra automaticamente na fila de espera.
   - Se o ID informado for o mesmo do aluno que já está no banheiro: O sistema registra o seu retorno (calculando a duração em minutos) e já puxa o próximo aluno da fila (se houver) para o status ativo.
d) Atualização em Tempo Real (GET): O painel frontend (`js/app.js`) faz consultas automáticas a cada 10 segundos para a API para buscar o status atual ("Fora da sala agora", "Fila do Banheiro" e "Registros do dia") e atualiza a tela para todos que estiverem visualizando.


3. ESTRUTURA DOS ARQUIVOS E SUAS RESPONSABILIDADES
----------------------------------------------------------------------

[ CONFIGURAÇÃO E BANCO DE DADOS ]
- config/database.php: Responsável por estabelecer a conexão com o banco de dados MySQL via PDO. Define constantes de acesso e gerencia exceções de conexão.

[ MODELOS (MVC - Camada de Dados) ]
- models/Aluno.php: Classe contendo as queries relacionadas aos dados dos alunos. Possui métodos como `getAlunoById($id)` para validação de existência.
- models/Registro.php: Coração do banco de dados. Contém métodos para buscar quem está no banheiro (`getAtivo()`), adicionar à fila (`entrarFila()`), buscar histórico do dia (`getRegistrosHoje()`), registrar a saída (`registrarSaida()`), registrar a volta e o tempo gasto (`registrarRetorno()`), e puxar o próximo da fila (`proximoDaFila()`).

[ ROTAS / API ]
- api/registro.php: O endpoint principal do sistema. Gerencia os métodos POST e GET para a página de controle.
- api/dashboard.php: Provê dados agregados (totais, médias e rankings) para a página de estatísticas conforme requisições GET.
- api/calendario.php: Filtra registros por data (`?data=YYYY-MM-DD`) para consulta histórica.

[ FRONTEND ]
- index.html: Documento principal da operação diária.
- dashboard.html: Painel de indicadores e performance dos alunos.
- calendario.html: Interface de consulta de histórico retroativo.
- css/style.css: Design unificado para todo o sistema (variáveis CSS para cores, temas de cards e tabelas).
- js/app.js: Lógica do controle de saída (painel principal).
- js/dashboard.js: Lógica de processamento de estatísticas e ranking.
- js/calendario.js: Lógica de construção dinâmica do calendário e filtros de data.


4. DETALHES TÉCNICOS E MANUTENÇÃO (BACKEND)
----------------------------------------------------------------------
O banco de dados MySQL utiliza 3 tabelas (`alunos`, `registros_saida`, `fila_banheiro`).

[ Scripts de Manutenção ]
- fix_db.php: Script utilitário utilizado para atualizar a estrutura da tabela `registros_saida`, garantindo que o campo `status_alunos` aceite corretamente o ENUM ('EM_ANDAMENTO', 'CONCLUIDO') e limpando registros órfãos ou inválidos.

[ Lógicas SQL Avançadas (models/Registro.php) ]
- Cálculo de Duração: Feito via `TIMESTAMPDIFF(MINUTE, hora_saida, NOW())` durante o update de retorno.
- Estatísticas do Dia: Uso de funções agregadas `COUNT(DISTINCT ...)`, `AVG(...)` e `SUM(...)` para gerar o resumo do dashboard em uma única query.
- Ranking de Tempo: Agrupamento (`GROUP BY`) por aluno com ordenação decrescente por tempo total gasto.


5. ESPECIFICAÇÕES: PÁGINA DE DASHBOARD (ESTATÍSTICAS)
----------------------------------------------------------------------
Fornece métricas em tempo real (atualização a cada 15s) para monitorar o uso do banheiro.

[ Indicadores (Cards) ]
- Total de Saídas: Contagem absoluta de acessos no dia atual.
- Alunos Distintos: Quantidade de alunos únicos que saíram (evita duplicidade do mesmo aluno).
- Tempo Médio: Valor médio em minutos de todas as sessões finalizadas do dia.
- Tempo Total: Somatório global de tempo de uso do banheiro no dia.

[ Listagem e Ranking ]
- Relatório de Alunos: Tabela dinâmica listando nome, frequência (quantidade de vezes) e tempo acumulado individual.
- Top Demorados: Lista visual ranqueada pelas maiores durações acumuladas, destacando os 3 primeiros com badges estilizados de ranking.


6. ESPECIFICAÇÕES: PÁGINA DE CALENDÁRIO (HISTÓRICO)
----------------------------------------------------------------------
Permite a visualização detalhada de qualquer dia do mês ou ano selecionado.

[ Funcionamento do Calendário ]
- Navegação: Controle de meses e anos com preenchimento dinâmico de datas.
- Seleção de Data: Ao clicar em um dia específico, o sistema dispara uma requisição para a `api/calendario.php` enviando o parâmetro ISO da data.
- Detalhamento: Apresenta uma tabela com todos os nomes, horários de saída/retorno e durações exclusivas daquele dia pesquisado. Se um registro não possui retorno no banco, a interface exibe o status como "Em aberto".


7. DETALHAMENTO INDIVIDUAL DE ARQUIVOS
----------------------------------------------------------------------

[ RAIZ DO PROJETO ]
- index.html: Interface principal de controle de entrada/saída. Contém o formulário de ID e as visualizações de quem está fora, fila e registros do dia.
- dashboard.html: Página de análise de dados com cards de estatísticas, tabela de frequência e ranking de uso por tempo.
- calendario.html: Página de consulta retroativa que permite visualizar registros de qualquer data selecionada em um calendário interativo.
- fix_db.php: Script para limpeza e correção estrutural do banco de dados (ajuste de ENUMs e remoção de registros nulos).
- db_test_debug.php: Script de teste rápido para validar a conexão com o banco e listar as colunas das tabelas, usado para debugar erros de SQL.
- schema.json: Representação estrutural (metadados) das tabelas do banco de dados em formato JSON para referência do sistema.
- cols.json: Mapeamento de colunas e tipos de dados utilizados para validação interna.

[ PASTA: config/ ]
- database.php: Configuração central da conexão PDO. Define host, usuário, senha, banco e porta. É incluído em todos os arquivos que acessam o banco.
- test_db.php: Script de diagnóstico de conexão com configurações alternativas (útil para testar portas diferentes como 3307).

[ PASTA: models/ ]
- Aluno.php: Representa a entidade Aluno. Contém o método `getAlunoById()` para buscar dados pessoais na tabela `alunos`.
- Registro.php: A classe mais complexa do sistema. Gerencia toda a lógica SQL de registros de saída, fila de espera, cálculos de tempo, estatísticas de dashboard e filtros de calendário.

[ PASTA: api/ ]
- registro.php: Endpoint que processa entradas/saídas e fornece os dados para a página principal (`index.html`).
- dashboard.php: Endpoint que processa e retorna as estatísticas do dia para a `dashboard.html`.
- calendario.php: Endpoint que filtra e retorna registros de uma data específica para a `calendario.html`.
- buscar_aluno.php: Mini-API que retorna apenas o nome de um aluno dado seu ID, usada para validações rápidas.

[ PASTA: js/ ]
- app.js: Script principal. Gerencia eventos de botões, temporizador do aluno ativo, requisições Fetch para registros e atualização automática da UI da home.
- dashboard.js: Gerencia a formatação de tempos (h/min), requisições de estatísticas e renderização dinâmica de cards, tabelas e rankings.
- calendario.js: Contém a lógica de construção do grid de calendário, troca de meses/anos e fetch de detalhes por dia.

[ PASTA: css/ ]
- style.css: Folha de estilos unificada. Utiliza variáveis CSS para padronizar a paleta de cores (Bege, Azul Marinho, Azul Suave) e garantir um design premium e responsivo em todas as páginas.

[ PASTA: scriptDB/ ]
- scriptEntradaESaida.sql: Script SQL completo para criação do banco de dados `sistema_banheiro`, incluindo as definições de tabelas, chaves estrangeiras e a inserção inicial da lista de alunos da turma.
- .htaccess: Arquivo de configuração do servidor para proteção de diretórios ou regras de roteamento (se aplicável).
